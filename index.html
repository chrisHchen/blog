需要解决的问题

和单纯前端js渲染的相同点:

  • css/less/sass
  • css module
  • routing
  • code splitting

和单纯的前端js渲染相比, ssr 需要处理下面几个额外的事项：

  • isomorphic render
  • data prefetching
  • matching hydration

如何在不使用现有 ssr 框架的情况下实现上述需求

  • two sets of webpack configration, one for client, one for server.
  • two sets of entries respectively.
  • use css-loader/locals for css module at server side. It doesn’t embed CSS but only exports the identifier mappings.
  • async/await data prefetching
  • use babel-node to support es6 syntax.

示例代码
react-ssr-basic

示例代码的目标是在使用最少类库的情况下，构建一个基本的 ssr 例子, 还原我们经常用到的 ssr 框架到底为我们做了什么。

例子中用到了包括 webpack, babel, css-loader, less-loader, postcss-loader 等常用的构建类库。 另外就是基本的 react 和 react-router。 http 服务器选择使用 koa。可见使用的基本都是纯前端渲染也都会用到的类库。

preact 源码(总结篇)

diff 做了什么？

用作者的原话就是:

Apply differences in a given vnode (and it’s deep children) to a real DOM Node.

其中的 differences 本质上其实是 component 实例的 base 属性和 component 实例的 render 方法执行后返回的 vnode 之间的 difference。 其中base 属性就是上一次 diff 后生成的 Dom 节点。 所以就是用上次的 dom 和本次的 vnode 做 difference/diff。两种状态作比较，通过 diff 方法内的一整套规则，用最小的代价来做更新。

preact 源码(4)

buildComponentFromVNode方法

这是上篇没有展开的一个方法。
preact 通过 buildComponentFromVNode(dom, vnode, context, mountAll) 进行 component diff。

组件是 react/preact 的基本思想。所有的组件都会通过 buildComponentFromVNode 方法来初始化 dom 或者diff dom。 buildComponentFromVNode 方法决定了本次 diff 是重用已有的 component 还是新建 component。

第一个参数 dom 是需要 diff 的 dom 节点。第二个参数是对应的 vnode。这里的 vnode.nodeName 为 component 的构造函数。

方法首先通过 vnode.attributes 和 vnode.nodeName.defaultProps 获得 component 实例的 props 参数。
