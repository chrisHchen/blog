draft-js-plugin 源码

draft-js 是 facebook 推出的基于 React 和 immutable 数据模型的富文本编辑器构建框架。(文档)

draft 通过改变 Editor 组件的 editorState prop 来渲染每次内容改变的结果。我们可以在 onChange 事件内生成新的 editorState 对象(editorState 是 immutable 的，只能生成新的，不能修改)，来改变 Editor 的 editorState。也可以通过给 Editor 组件传入各种 prop 来改变他的默认行为，生成符合我们要求的 editorState。

而 draft-js-plugins 则提供了一种插件机制，将每种独立行为都做成一个插件，用配置插件的方式来灵活的生成符合需求的富文本编辑器。

fastdom

浏览器渲染一个页面通常需要如下的步骤:

其中的 layout 有时候也称为 reflow。

layout是非常消耗性能和时间的，所以应该尽量避免js的执行产生 layout 的情况。特别是在有动画的情况下，太多的layout会使得浏览器的刷新频率低于60fps，出现动画卡顿的现象。具体的原理和细节可以参考这篇文章:

Avoid Large, Complex Layouts and Layout Thrashing

fastdom 就是为了优化 layout，防止出现 forced synchronous layouts 的一个js库。

源码比较简单，首先 fastdom 库是一个全局单例的库:

通过管理单例上的 reads 和 writes 两个数组来缓存读(measure)写(mutate)两种操作

通过 scheduleFlush 函数在下一帧(requestAnimationFrame)执行数组中的函数 — 先执行 reads 再执行 writes。

这样所有的读写操作都被分到不同的数组中，然后异步的统一执行，保证 js执行过程中不出现 forced synchronous layouts 的现象。
