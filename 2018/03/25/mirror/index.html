mirrorx 源码【v0.2.10】

Mirror 是一款简洁、高效、易上手的 React 框架。

  • doc-cn
  • api-cn

通过文档可以感觉出 mirror 最主要是简化了 redux 概念中的 actionType, 框架会自动生成 actionType, 而使用者要做的只是调用 mirror 提供的 actions对象上相应的方法就可以。比如: actions.A.setAge(10)。 这样也同时简化了 thunk , 直接在 effects 中调用 actions 上的方法就可以了。

下面简单的看下源码的实现过程：

models

传入 model 的对象会进入 addActions(m.name, m.reducers, m.effects) 方法，将 reducers 放到 namespace 为 m.name 的全局 actions 对象上。effects 也会添加到一个全局的 effects 对象上，但这个对象不对外暴露:

options.addEffect(${modelName}${SEP}${effectName}, effects[effectName])
后续会在 middleware 中调用这些方法。

actions

可见，actions.modelName 对象上的方法只需要一个 payload 参数就可以了， type 默认都是 ${modelName}${SEP}${actionName}, 其中 SEP='/'。

routerMiddleware 处理更新 history location 的 api 方法(push, replace, go…), 这里集成了 react-router-redux。

router

最后剩下 router 对象。这个包装后的router主要做了两件事情：
1.将改变 location 的那些方法添加到 actions.routing 对象上。
2.创建 history。
3.关联 routing history 到 store（使用 react-router-redux 的 ConnectedRouter 组件 ）。
