buildComponentFromVNode方法

这是上篇没有展开的一个方法。
preact 通过 buildComponentFromVNode(dom, vnode, context, mountAll) 进行 component diff。

组件是 react/preact 的基本思想。所有的组件都会通过 buildComponentFromVNode 方法来初始化 dom 或者diff dom。 buildComponentFromVNode 方法决定了本次 diff 是重用已有的 component 还是新建 component。

第一个参数 dom 是需要 diff 的 dom 节点。第二个参数是对应的 vnode。这里的 vnode.nodeName 为 component 的构造函数。

方法首先通过 vnode.attributes 和 vnode.nodeName.defaultProps 获得 component 实例的 props 参数。

该方法对 component 做 diff 的逻辑如下:

  1. 当 dom 存在， 并且 dom._component 或者其祖先 component 的构造器 == vnode.nodeName 时， 认为 vnode 是 dom 的 owner, 此时复用之前的 component 实例，并调用 setComponentProps 方法赋予新的 props.

  2. 如果 dom._component 存在，但和 vnode.nodeName 构造器不一致，则认为不是同个 component。此时进行 unmountComponent 操作并回收 component 实例。 然后调用 c = createComponent(vnode.nodeName, props, context) 方法构建新的 component 实例, 并调用 setComponentProps(c, props, SYNC_RENDER, context, mountAll); 方法，此处是新创建的实例所以传递了 SYNC_RENDER 来直接 render (instead of enqueue)。

如果 dom 有变化则移除 dom, 最后返回 dom = c.base

renderComponent方法

该方法决定了 component 的更新策略，以及对 HOC 的处理策略。通过 render 产生的 vnode tree 来调用 diff 算法，生成或更新 dom tree。

其中 isChild 属性指示 component 是否是 HOC 的 root component 实例。

首先通过 component.base 判断是是首次 mount 还是 update，因为通过 createComponent 方法新创建的 component 实例是没有 base 属性的。

如果是 update 并且 opt !== FORCE_RENDER, 则执行 shouldComponentUpdate 周期函数, 如果返回true，则在执行 componentWillUpdate 周期函数。

执行完上面两步后会将所有的 prev 置空，并认为此时的 component 已经是和 update 前的状态无关了，是 clean 的。可知如果在接下来的 render 方法里调用 this.props 则返回的是新的 prop 了。

如果 shouldComponentUpdate 返回 true， 接下里执行 rendered = component.render(props, state, context) 方法。 然后将context 和 component.getChildContext() 做 merge 后作为新 context 传给子 component。

接下来对 component 是否是 HOC 分别做判断:

如果存在 HOC(typeof rendered.nodeName === 'function') 则进行 HOC diff，逻辑如下:

  1. 如果 component._component 存在，且 component._component 的构造器等于 rendered.nodeName，且 key 也一致(childProps.key==component._component.__key),则认为是同一个 component 的更新操作，所以调用 setComponentProps 更新, 并直接render。

  2. 否则认为是不同的 component，所以通过 createComponent 创建 component 实例
